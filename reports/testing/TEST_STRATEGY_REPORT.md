## 2. 关键测试场景识别

### 2.1 题目管理核心场景
| 场景 | 优先级 | 测试要点 |
|------|--------|----------|
| 题目 CRUD | P0 | 五核心字段校验、选择题答案 A/B/C/D 格式转换 |

### 2.2 AI 功能场景
| 场景 | 优先级 | 测试要点 |
|------|--------|----------|
| 预备题目入库 | P0 | 状态流转 pending→approved、答案格式验证 |

- 选项为空/超长/含空字符串
- 答案索引超出选项范围
- 分类/标签不存在时的级联处理
- 并发创建题目的数据一致性

## 4. 自动化测试建议

### 4.1 推荐技术栈
| 层级 | 工具 | 理由 |
|------|------|------|
| 单元测试 | `pytest` + `pytest-asyncio` | 与现有项目兼容 |
| API 测试 | `httpx` + `pytest` | 异步支持、FastAPI 友好 |
| E2E 测试 | `Playwright` | 多浏览器、Web 界面验证 |
| 覆盖率 | `pytest-cov` | 生成 HTML 报告 |

### 4.2 目录结构建议
```
tests/
├── unit/              # 单元测试
│   ├── test_models.py
│   └── test_services.py
├── integration/       # 集成测试（已有）
│   └── test_approve_staging.py
├── api/               # API 测试
│   ├── test_questions_api.py
│   ├── test_categories_api.py
│   └── test_tags_api.py
├── e2e/               # E2E 测试（空）
│   └── test_web_workflow.py
└── conftest.py        # 共享 fixture
```

### 4.3 CI 集成建议
```yaml
# .github/workflows/test.yml
- 预提交：pytest tests/unit -q
- PR 检查：pytest tests/api --cov=core --cov-report=html
- 夜间构建：pytest tests/e2e --headed
```
